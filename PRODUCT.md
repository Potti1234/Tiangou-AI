# Product

## Register

product

## Users

Grid researchers, infrastructure analysts, and engineers inspecting Hong Kong electricity-grid topology, demand assumptions, solver handoff quality, and stress-scenario behavior. They use the app as an analysis dashboard while validating public-data-derived models, not as an operational control room.

## Product Purpose

Tiangou-AI builds an auditable Hong Kong grid model from OpenStreetMap and public assumption tables, exposes FastAPI endpoints for topology and solver handoff inspection, and provides a frontend for reviewing provenance, confidence, validation warnings, and scenario outputs. Success means users can see what is observed, inferred, or synthetic before relying on model outputs.

## Brand Personality

Technical, transparent, restrained. The interface should feel like a serious analysis tool: dense enough for repeated expert use, careful about uncertainty, and direct about limitations.

## Anti-references

Avoid marketing-page composition, decorative dashboard chrome, glassmorphism, oversized hero sections, hidden assumptions, and any presentation that implies operational certainty or real blackout training data where the model is assumption-driven.

## Design Principles

- Put provenance near every claim.
- Prefer compact, comparable panels over presentation-heavy sections.
- Make synthetic and inferred assumptions visible in both copy and data displays.
- Use familiar product UI controls for simulation workflows.
- Keep topology, analytics, solver, and dynamic-simulation surfaces consistent.

## Accessibility & Inclusion

Use readable contrast, keyboard-reachable controls, clear loading and error states, reduced-motion-safe transitions, and redundant labels for status colors. Treat dense technical content as scannable product UI rather than decorative visualization.
